@Knut Wenzig wow, this is awesome! A few points.

  • https://iqss.github.io/dataverse-installations/data/data.json is used as a data source for https://hub.dataverse.org (see here in the code for the hub). It's still completely valid to look at that upstream data.json as the source of truth for the list of installations but it gets massaged into https://hub.dataverse.org/api/installations which we would like people to switch to, eventually. @Julian Gautier is the primary maintainer of data.json via the script at https://github.com/IQSS/dataverse-installations . @Juan Pablo Tosca Villanueva is the primary developer of the hub, and the code is https://github.com/IQSS/dataverse-hub
  • There's a suggestion to incorporate re3data at https://github.com/IQSS/dataverse-installations/issues/190 and https://groups.google.com/g/dataverse-community/c/xFRXZ7BAuQA/m/8xH_iPTqBQAJ if you'd like to link to this Zulip topic (or I can).
  • I'm glad to hear the re3data team is so responsive. I've been aware of the service for years but haven't interacted with it much apart from looking at the occasional listing.

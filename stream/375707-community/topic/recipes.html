Jan Range (Mar 28 2025 at 15:49):

Should we make exceptions for languages that are strictly non-snake-case? This is the new section in the Readme:

Naming conventions

  • File names should be in lowercase and use underscores to separate words (e.g. create_dataset.py)
  • Directory names should be in lowercase and use underscores to separate words (e.g. python_create_dataset)

Note: If a language convention requires it, use camel/kebab/pascal case, but make sure to align with the existing naming conventions. Examples are:

  • JavaScript: Use camel/kebab case (e.g. createDataset.js or create-dataset.js)
  • Java: Use pascal case (e.g. CreateDataset.java)
